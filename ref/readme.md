一、项目总览与团队分工
1.1 项目核心定位
本项目以研究生第二党支部 2026 年支部品牌需求为核心，打造 “党小二” 智能助手，依托 AI 大模型与 Agent 技术，整合党建权威知识与高研院研究生全流程业务体系，实现 7×24 小时智能问答、办事指引、政策解读等能力，既减轻支部重复工作负担，也为研究生群体提供高效、合规、贴心的党建 + 学业双向服务。

三、详细技术方案
3.1 UI 设计与前端开发方案
3.1.1 核心设计原则
1.	风格定位：庄重合规的党建基调 + 年轻化的研究生适配风格，主色调采用党建红 + 中性白，视觉简洁大气，操作便捷高效
2.	适配要求：响应式布局，同时适配 PC 端、移动端（微信 / 企业微信内嵌、手机浏览器），覆盖研究生全场景使用需求
3.	合规要求：严格遵守党建宣传视觉规范，杜绝不当设计元素，所有内容符合高校党建工作要求
3.1.2 技术栈选型
技术模块	选型推荐	选型优势
核心框架	Vue 3 + TypeScript	上手门槛低，中文生态完善，TypeScript 保障代码稳定性，减少开发 bug
UI 组件库	Ant Design Vue	成熟稳定，组件丰富，支持自定义主题，可快速适配党建视觉规范
状态管理	Pinia	Vue 3 官方推荐，轻量易用
工程化工具	Vite	构建速度快，开发体验好，大幅提升开发效率
版本管理	Git + Gitee/GitLab	实现团队协作开发，版本回溯，代码管控
3.1.3 核心页面与功能模块
1.	首页：党建宣传 banner、核心功能快捷入口（智能问答、办事指南、知识库、热门问题）、通知公告、用户快捷操作区
2.	智能对话核心页：多轮对话界面、上下文记忆展示、问题分类标签、历史对话记录、答案来源溯源、问题反馈入口
3.	办事指南专区：分为党建办事、研究生业务两大板块，结构化展示流程节点、所需材料、时间节点、注意事项，支持一键跳转智能咨询
4.	知识库检索页：支持全文检索、分类筛选（党建类 / 业务类）、文档预览、内容收藏、一键引用
5.	个人中心：历史对话、收藏内容、反馈记录、个人信息、权限管理入口
6.	时政要闻：摘录的时政要闻
7.	后台管理页：知识库管理、用户管理、对话日志查看、内容审核、系统配置、数据统计看板
3.2 核心AI Agent能力开发方案
3.2.1核心技术栈选型
技术模块	选型推荐	选型优势
基座大模型	Qwen2-7B/14B / ChatGLM3-6B	中文能力顶尖，开源可免费商用，支持本地化部署，7B/14B 模型对服务器算力要求低，适配高校服务器资源，支持场景化微调
嵌入模型	BGE-zh-large-v2	中文场景检索效果顶尖，适配党建 + 业务知识库的精准检索需求
向量数据库	Milvus（主选）/ Chroma（轻量版）	开源高性能，支持分布式部署，适配大规模知识库的向量存储与快速检索
Agent 框架	LangChain + LlamaIndex	成熟开源生态，快速搭建 RAG 检索与工具调用能力，大幅减少重复开发
内容审核	开源中文内容审核模型 + 自定义党建敏感词库	实现用户输入、模型输出双重审核，100% 保障内容合规
3.2.2 核心 Agent 功能模块
1.	双领域 RAG 知识库检索系统
1)	党建知识库：收录党章党规、党的最新政策文件、党务工作指南、入党全流程规范、组织生活制度、党史学习材料等官方权威内容，确保来源可溯、内容准确。
2)	研究生业务知识库：收录高研院培养方案、选课开题规则、论文投稿规范、奖助金申请政策、毕业离校流程、校园办事指南等学院官方发布的全量业务文件，覆盖研究生全培养周期。
3)	知识库更新机制：支持后台一键上传文档、自动分块向量化，建立 “专人更新 + 双人审核” 的月度更新机制，确保内容时效性与权威性。
2.	智能问答与多轮对话引擎
1)	支持多轮对话、上下文记忆、问题自动分类，精准区分党建咨询、研究生业务咨询、通用咨询三类场景，适配不同的回答策略。
2)	实现答案溯源功能，所有核心内容均标注引用来源，可一键跳转对应原文，解决大模型幻觉问题。
3)	支持多场景问答适配：政策解读、流程指引、问题答疑、模板推荐、时间提醒等。
3.	工具调用模块
1)	内置办事流程查询、时间节点提醒、官方文档模板下载、政策文件溯源、相关办事入口跳转等工具能力。
2)	可扩展功能：入党进度查询、培养节点提醒、论文投稿期刊匹配、党建学习打卡等特色工具。
4.	双重内容安全审核体系
1)	前置审核：用户输入内容实时审核，过滤敏感、违规、不当提问。
2)	后置审核：模型生成内容二次审核，叠加党建合规规则校验，杜绝错误政治表述、不合规内容，确保所有输出 100% 符合党建工作要求。
3.3 后端服务与部署方案
3.3.1技术栈选型
技术模块	选型推荐	选型优势
后端框架	FastAPI	Python 原生生态，与 AI 大模型、LangChain 无缝衔接，开发效率高，性能优异，自动生成 API 文档，适配团队协作
业务数据库	PostgreSQL / MySQL	开源稳定，适配结构化数据存储，支撑用户信息、对话记录、权限配置等数据管理
缓存数据库	Redis	缓存高频问答、用户会话、向量嵌入结果，大幅提升系统响应速度
容器化部署	Docker + Docker Compose	轻量易维护，适配小团队运维，实现 “一次打包，处处运行”，降低部署成本
监控运维	Prometheus + Grafana	实时监控系统性能、接口响应时间、服务器资源使用率，配置告警机制，保障系统稳定运行
3.3.2 核心后端服务模块
1.	用户认证与权限服务：采用 JWT 令牌认证，对接学校统一身份认证（CAS / 企业微信），实现校内账号一键登录；氛围 “普通用户 / 管理员” 两级，无需复杂的角色体系，内部使用可甚至省略登录环节，或仅设置简单的访问密码
2.	对话核心服务：对接 Agent 引擎，实现会话管理、对话分发、上下文记忆管理、历史对话存储等功能。
3.	知识库管理服务：实现文档上传、解析、分块、向量化、向量存储、检索调度、内容更新全流程管理。
4.	接口服务：前端页面所需的基础数据接口
5.	日志与审计服务：全操作日志留存，对话记录可追溯，满足党建工作合规审计要求。
3.3.3 部署方案
1.	部署环境：优先选择学校校内服务器本地化部署，实现数据不出校，杜绝数据外泄风险，符合高校数据安全要求；备选国内合规云服务器（阿里云 / 腾讯云高校专项）。
2.	高可用保障：采用主从架构部署，核心服务多副本冗余，实现 7×24 小时高可用。
